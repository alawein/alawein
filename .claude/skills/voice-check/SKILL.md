# Voice Check

Lint the target against `alawein/docs/style/VOICE.md`. Classify every finding
as BLOCKING or ADVISORY. Report only real violations — no false positives, no
style opinions beyond the contract.

## How to invoke

```
/voice-check path/to/file.md
/voice-check (paste text inline after the command)
```

## BLOCKING violations

These fail governed surfaces (README, CLAUDE.md, AGENTS.md, prompt kits, docs,
bios, LinkedIn copy):

**1. Forbidden register** — flag any occurrence of:
`passionate about`, `leveraging`, `innovative`, `cutting-edge`, `excited to`,
`it's worth noting`, `interestingly`, `delve into`, `deep dive`, `utilize`,
`in today's world`, `moving forward`, `synergy`, `holistic`, `game-changing`,
`groundbreaking`, `seamlessly`, `robust solution`, `empower`, `transformative`,
`I am pleased to`, `I hope this helps`, `feel free to`, `as mentioned above`,
`next-gen`, `world-class`, `powerful`, `future-ready`

**2. Banned identity terms** — flag:
`Morphism Systems`, `morphism-systems`, `morphism.systems`, `@morphism/`,
`gainboy`, `neper`, `qaplibria`, `epistemic-stack`, `reasonbench`

**3. Stale personal claims** — flag:
`PhD candidate`, `Ph.D. candidate`, `finishing a PhD`, `finishing my PhD`,
`pursuing a PhD`, `pursuing my PhD`, `ABD`

**4. Em dash overuse** — count em dashes (—) per paragraph. Flag any paragraph
with 3 or more. Report paragraph excerpt and count.

**5. Fragment sentences** — flag any sentence lacking a subject or verb that is
not a heading, list item label, or code block.
Examples to flag: `"Multi-round structured probing."` `"Deterministic scoring."`

**6. AI attribution** — flag:
`Generated with Claude`, `Co-Authored-By: Claude`, `Generated by Claude`,
`AI-generated with`, `AI-generated by`

## ADVISORY violations

**7. Low-signal comment language** — flag in code comments only:
`simply`, `obviously`, `easy`, `easily`, `magic`, `hack`

**8. Em dash as cause-effect connector** — flag sentences where an em dash
connects two independent clauses (neither side is a parenthetical insertion).
Pattern: `[complete clause] — [complete clause]`.
Report with suggested colon replacement.

**9. Passive voice** — flag:
`it was found`, `it can be seen`, `it has been shown`, `it is noted that`,
`it should be noted`

## What NOT to flag

- Technical terms that contain flagged words in domain context (`robust` in a
  stability proof; `leverage` in a mechanics context)
- Quoted text or examples demonstrating bad patterns
- Code strings, variable names, and identifiers
- HTML comments `<!-- -->`
- Rules 7 (low-signal comments) applied outside code comment blocks

## Output format

```
VOICE CHECK — [filename or "inline"]
Checked: [date]

BLOCKING ([n] found)
────────────────────────────────────────
[LINE n] [rule]: "[offending text]"
  Fix: [one-line correction]

ADVISORY ([n] found)
────────────────────────────────────────
[LINE n] [rule]: "[offending text]"
  Fix: [one-line correction]

RESULT: PASS | FAIL ([n] blocking)
```

If no violations:

```
VOICE CHECK — [filename]
RESULT: PASS — no violations found
```
