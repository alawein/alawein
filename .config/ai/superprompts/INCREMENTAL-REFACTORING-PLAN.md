# Incremental Refactoring Plan Prompt

## Objective
Plan incremental refactoring to consolidate systems without disruption.

## Planning Framework

### Refactoring Phases
- Phase 1: Low-risk items
- Phase 2: Medium-risk items
- Phase 3: High-risk items
- Phase 4: Integration
- Phase 5: Optimization

### Refactoring Priorities
- Impact assessment
- Effort estimation
- Risk evaluation
- Dependency analysis

### Refactoring Timeline
- Phase duration
- Milestone dates
- Validation points
- Rollback triggers

### Refactoring Validation
- Test coverage
- Performance validation
- Functionality verification
- Integration testing

## Strategy

### Incremental Approach
- Small, manageable steps
- Frequent validation
- Early feedback
- Continuous improvement

### Risk Management
- Risk identification
- Risk mitigation
- Contingency planning
- Rollback procedures

## Output Format

### Refactoring Plan
- Phases and timeline
- Priorities and sequencing
- Effort estimates
- Risk assessments
- Validation strategy
- Rollback procedures

### Metrics
- Refactoring items: [count]
- Phases: [count]
- Timeline: [weeks]
- Risk level: [Low/Medium/High]

## Success Criteria
- [ ] Phases defined
- [ ] Priorities set
- [ ] Timeline established
- [ ] Risks identified
- [ ] Validation planned
