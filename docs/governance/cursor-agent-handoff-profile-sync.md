## 2. Revision and enhancements plan

Execute in order when improving or extending the setup.

### Phase A — Consistency and hygiene

1. **Replace any remaining “guides” wording** in alawein with “_pkos (formerly guides)” or “_pkos” where it refers to the PKOS repo. Check: `profile-sync-from-guides.md`, Files table (“Generated by guides” → “Generated by _pkos”), Quality gates (“guides or the exported YAML” → “_pkos or the exported YAML”).
2. **_pkos templates/README.md** — Update “guides root” to “_pkos root” and “alawein/alawein” references so they stay correct after rename.
3. **Alawein doc contract** — If `scripts/validate-doc-contract.sh` or `docs/governance/documentation-contract.md` still mention `README-backup-20250807.md` or `docs/README-backup-20250807.md`, update to `README.md` and `docs/README.md` only.
4. **Deprecate `docs/README-backup-20250807.md`** — Keep as a short stub that points to `../README.md` and `docs/README.md`; or remove and add a redirect in docs index if the repo supports it.

### Phase B — Export and sync robustness

5. **Export script default path** — In _pkos `scripts/export-profile.py`, make the default alawein path configurable (e.g. env `ALAWEIN_REPO_PATH` or a small config file) so it works when _pkos is at `Desktop/_pkos` or when moved next to alawein.
6. **PyYAML in alawein** — Document that `sync-readme.py` needs PyYAML when `profile-from-guides.yaml` is present. Optionally add `pyyaml` to a requirements or optional-deps note in README/CONTRIBUTING so CI can install it when running profile sync.
7. **Sync check behavior** — Ensure `sync-readme.py --check` only fails on profile section when `profile-from-guides.yaml` exists; when the file is missing, do not require PyYAML or profile block changes.

### Phase C — meshal-web and LinkedIn

8. **meshal-web** — Add a build step or script that reads `profile-export.yaml` (or a copy dropped into meshal-web) and injects hero/bio/CTA from the export; or document manual steps. Update meshal-web README to state profile content source.
9. **LinkedIn snippet** — In _pkos, extend `export-profile.py` to write `out/linkedin-snippet.md` (or similar) with `linkedin_headline` and a truncated `about` for copy-paste. Document in _pkos `templates/README.md`.

### Phase D — Optional structural improvements

10. **_pkos move** — If user moves _pkos to `../GitHub/github.com/_pkos`, update HANDOFF-CODEX key paths and export default; set `ALAWEIN_REPO_PATH` if needed.
11. **Notion / CV** — Add `templates/notion.yaml` or `templates/cv-master.yaml` and optional export artifacts if Notion or master CV should be driven from the same export.

## 3. Syncing and maintenance runbook

### Regular sync (after editing profile or copy in _pkos)

```bash
# 1. In _pkos (path may be Desktop/guides or Desktop/_pkos)
cd C:\Users\mesha\Desktop\guides   # or ...\Desktop\_pkos
python scripts/export-profile.py --write-alawein
# Optional: set ALAWEIN_REPO_PATH to alawein repo root if default path is wrong

# 2. In alawein
cd C:\Users\mesha\Desktop\GitHub\github.com\alawein\alawein
python scripts/sync-readme.py
```

### After editing `projects.json` in alawein

```bash
cd C:\Users\mesha\Desktop\GitHub\github.com\alawein\alawein
python scripts/sync-readme.py
```

### Quality gates (alawein, before commit/PR)

```bash
python scripts/sync-readme.py --check
./scripts/validate-doc-contract.sh --full
```

### Rename guides → _pkos (once, when folder not in use)

```powershell
Rename-Item -Path "C:\Users\mesha\Desktop\guides" -NewName "_pkos"
```

Then update any hardcoded `guides` paths in HANDOFF-CODEX and export script default if necessary.
