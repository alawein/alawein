# Profile sync from guides

**Purpose:** Document how the README About section and profile copy stay in sync with the PKOS repo (Desktop/_pkos, formerly guides), which is the single source of truth for positioning, resume, and profile copy.

**For Cursor agents:** For revision and enhancements (phases, runbook, checklist), read and follow [cursor-agent-handoff-profile-sync.md](cursor-agent-handoff-profile-sync.md).

## Source of truth (_pkos repo, formerly guides)

- **Profile record:** `db/profile/meshal-alawein.md` (YAML frontmatter).
- **Copy (tagline, bios, headline):** `docs/career/profile-copy.yaml`.
- **Export:** `scripts/export-profile.py` produces `out/profile-export.yaml` and can write `profile-from-guides.yaml` into this repo.

## Flow

1. **In _pkos (formerly guides):** Edit `db/profile/meshal-alawein.md` and/or `docs/career/profile-copy.yaml`, then run:
   ```bash
   cd /path/to/_pkos
   python scripts/export-profile.py --write-alawein
   ```
   This writes `profile-from-guides.yaml` into `alawein/alawein` (default: `../GitHub/github.com/alawein/alawein/`). Set `ALAWEIN_REPO_PATH` to point at this repo if needed.

2. **In alawein:** When `profile-from-guides.yaml` is present, run:
   ```bash
   python scripts/sync-readme.py
   ```
   This regenerates the README.md block between `<!-- SYNC:PROFILE:START -->` and `<!-- SYNC:PROFILE:END -->` from the YAML. If the file is missing, the profile block is left unchanged.

## Files

| File | Role |
|------|------|
| `profile-from-guides.yaml` | Generated by `_pkos`; gitignored in alawein. Do not edit by hand; re-export from `_pkos` to update. |
| `README.md` | Contains `SYNC:PROFILE` section; overwritten by `sync-readme.py` when profile YAML exists. |
| `scripts/sync-readme.py` | Reads `profile-from-guides.yaml` (if present and PyYAML available) and updates README.md profile block. |

## Quality gates

- `python scripts/sync-readme.py --check` verifies both projects sync (from `projects.json`) and profile sync (from `profile-from-guides.yaml` when that file exists). If the profile file is present but PyYAML is not installed, `--check` exits 1 and reports the missing dependency.
- **PyYAML:** When `profile-from-guides.yaml` is present, `sync-readme.py` requires the `pyyaml` package (`pip install pyyaml`). For CI that runs profile sync, add PyYAML to requirements or optional deps.
- Profile sync is **optional**: CI and contributors who do not have _pkos or the exported YAML can still run `sync-readme.py --check`; it will only validate the profile section when `profile-from-guides.yaml` exists. When that file is missing, no PyYAML or profile validation is required.

## Guides location and naming (recommendation)

- **Canonical name:** `_pkos` (Personal Knowledge Operating System). Rename from `guides` when the folder is not in use: from Desktop, run `Rename-Item -Path "C:\Users\mesha\Desktop\guides" -NewName "_pkos"` (or close Cursor/explorer using the folder first).
- **Location:** Keep at `C:\Users\mesha\Desktop\_pkos` or move to be sibling to the alawein repo (e.g. `../GitHub/github.com/_pkos`). After moving, set `ALAWEIN_REPO_PATH` or the default in `_pkos/scripts/export-profile.py` so the export still writes to `alawein/alawein/profile-from-guides.yaml`.
