# Follow-up: voice-check ignore-sentinel support

## Why this isn't blocked

The voice guide refactor (`feat/voice-guide-source-first-refactor`) ships
`docs/style/voice-unified.md` containing the forbidden-register list as
documentation examples. That file is auto-exempted from voice-check by the
existing `is_generated()` detection in `scripts/doctrine/validate.py:131-133`
(matches `<!-- generated by` in the first 512 chars). The assembler's output
includes `<!-- Generated by scripts/ops/build_voice_unified.py from docs/style/.
Do not hand-edit. -->` near the top, so the file is silently skipped.

No skip-list change was needed for the refactor PR.

## What is still useful as future work

The auto-exempt only helps fully-generated files. Hand-written docs that
legitimately quote the forbidden register cannot use the same trick:

- This refactor's spec (`docs/internal/specs/2026-05-11-voice-guide-source-first-refactor-design.md`) quotes forbidden phrases in its critique
- Future post-mortems describing AI-generated drift
- Audit READMEs that name the patterns they detect
- Design-defense docs that contrast bad/good register

Today, all four of the above sit under `docs/internal/` (exempt from doctrine CI
per `alawein/CLAUDE.md`), so the issue is theoretical for now. It becomes real
the first time someone writes one of the above outside `docs/internal/`.

## Proposed implementation

Add ignore-sentinel support to `scripts/doctrine/validate.py`:

- Recognize `<!-- voice-check:ignore-start -->` and `<!-- voice-check:ignore-end -->`
  as scope markers
- In `add_match_violations` (or wherever forbidden-register scanning happens),
  track an "ignoring" flag per file. Toggle on `start`, off on `end`. Skip
  matches inside ignore regions.
- Add a unit test in `scripts/tests/` covering: nested sentinels not supported,
  unterminated start emits a warning at end-of-file, sentinels in code fences
  still count.

The `voice-workflow.md` source already wraps §25's repo-wide scan command in
the sentinel pair, so it doubles as a fixture for the future test.

## Not a blocker for the refactor PR

This note exists so the design intent ("sentinels are the target state, paths
are the stopgap" — voice guide refactor spec §5) isn't lost. The refactor PR
shipped without needing the stopgap because the auto-exempt covered it.
