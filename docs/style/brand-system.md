# Brand system

## Layers

### Org invariant

- Alawein mark geometry remains fixed.
- Monochrome variants must preserve silhouette and clearspace.
- Text on image must meet accessibility contrast requirements.

### Family layer

- Theme family controls color ramps, material cues, and banner mood.
- Shared families should feel related without forcing identical layout or illustration choices.

### Repo layer

- Each repo may choose accent gradients, banner copy, screenshots, and framing within its assigned family.
- Repo visuals must map back to the asset manifest declared in the catalog.

## Asset pack requirements

Web-facing repos should declare and maintain:

- Logo source
- Banner source
- Screenshot directory
- Theme family
- Social preview image
- Favicon status

## Screenshot guidance

- Use product-real screenshots over placeholder canvases where possible.
- Keep browser chrome, device frames, and cropping style consistent within a repo family.
- Annotate only when the annotation adds operational context.

## Banner guidance

- Prioritize legibility over visual density.
- Keep copy short and specific.
- Use one focal message per banner.

## Acceptable repo-level variation

- Product: polished marketing emphasis
- Tooling: clean and utilitarian
- Research: restrained and editorial
- Archive: minimal branded wrapper
