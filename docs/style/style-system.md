# Style system

`VOICE.md` is the canonical voice contract. This file remains as a
compatibility entrypoint for older references that still point at
`style-system.md`.

## Use these sources

- Canonical voice contract: [`VOICE.md`](VOICE.md)
- Canonical workspace prompt: [`../../prompt-kits/AGENT.md`](../../prompt-kits/AGENT.md)
- Canonical portfolio prompt: [`../../prompt-kits/PORTFOLIO.md`](../../prompt-kits/PORTFOLIO.md)
- Terminology and lint source: [`terminology-registry.yaml`](terminology-registry.yaml)

## Overlay summary

- Personal/profile: first-person is allowed when the surface is genuinely personal
- Product: lead with user value and operational reality
- Tooling: lead with commands, interfaces, and operator concerns
- Research: lead with the question, reproducibility, and constraints
- Archive: lead with retention reason and current status

## Enforcement model

- Blocking: README surfaces, prompt kits, `CLAUDE.md`, `AGENTS.md`, and docs voice
- Advisory: code comments, mathematical notation, and commit style
