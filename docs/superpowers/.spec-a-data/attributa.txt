--- FEATURE GAPS ---
1. NLP model never loads — analyzeText() returns hardcoded 0.5/0.8 stub. To deliver "AI detection," the HuggingFace Transformers pipeline (already in dependencies: @huggingface/transformers ^3.7.1) needs to actually instantiate and run GLTR/DetectGPT scoring against token distributions.
2. CrossRef API integration is fake — resolveDOI() returns stub metadata for known publisher prefixes. Real citation validation requires live CrossRef API calls or a fallback cache.
3. Attribution analysis has no vector similarity — the attribution confidence scores (computeAttributions in attributionApi.ts) presumably use Supabase but the actual similarity computation method is unclear. No embedding model or similarity algorithm is visible in code.
4. CWE scanning is Python-only and regex-based — staticScan.ts has 8 rules targeting Python code patterns. An attribution tool handling academic code should support more languages (TypeScript, Java, etc.) and use AST-based detection rather than regex.
5. No public demo mode — the Scan/Workspace/Results flow exists as complete code but requires authentication and project setup. A portfolio-level attribution tool needs a public demo path (sample document, pre-computed results) that visitors can access without signing up.
6. The resume tailor module in _hidden/ is complete and local-first — it is production-ready but disabled. Re-enabling it would immediately add a distinct, high-quality feature to the product.
