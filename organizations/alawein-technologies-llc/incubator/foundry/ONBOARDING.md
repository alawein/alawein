**Getting new people productive and happy on your Foundry team.**

Complete this with every new hire. Print it out!

## üìÖ Day 1: Welcome

**Goal:** Feel welcomed, understand the vision, start learning

### Morning (9-12)

- [ ] **9:00** Welcome & setup
  - Manager greets them in person/video
  - Tech setup verification
  - Office tour (if in-person)

- [ ] **9:30** Welcome call
  - Why we're excited they're here
  - Team intro (5 min each team member)
  - Quick culture overview
  - Q&A

- [ ] **10:30** Tech setup
  - Make sure laptop works
  - Email/Slack/GitHub access
  - Help with local setup (if engineer)
  - Test that everything is working

- [ ] **11:30** Read documentation
  - START_HERE.md (10 min)
  - PROJECT.md (10 min)
  - QUICKSTART_FIRSTTIME.md (15 min)
  - Understand what company is doing

**Deliverable:** Understand what Foundry is

### Afternoon (1-5)

- [ ] **1:00** Lunch (team treat!)
  - Welcome celebration
  - Informal chat

- [ ] **2:00** Role-specific documentation
  - If engineer: GETTING_STARTED.md
  - If product: PRIORITY_RANKING.md
  - If marketing: MARKETING_PLAYBOOK.md
  - If ops: FINANCIAL_MODEL.md

- [ ] **3:00** Meet your onboarding buddy
  - 1:1 coffee/call
  - They answer any questions
  - Invite them to dinner/lunch

- [ ] **4:00** End of day
  - How are you feeling? (quick check-in)
  - Tomorrow's plan
  - Go home early! (you're tired)

**Deliverable:** Feel part of team, understand role direction

**Goal:** Understand codebase/process, start contributing

### Engineer Onboarding (Week 1)

**Day 2:**
- [ ] Setup dev environment (GETTING_STARTED.md)
- [ ] Run tests locally
- [ ] Read architecture docs
- [ ] Review code style
- [ ] Ask 5 "why" questions

**Day 3:**
- [ ] Make first code change (non-critical)
- [ ] Submit first pull request
- [ ] Get code review feedback
- [ ] Fix review feedback
- [ ] Celebrate first PR merged! üéâ

**Day 4:**
- [ ] Fix 2-3 bugs (small ones)
- [ ] Read 5 key code files

- [ ] Take notes on confusing parts

- [ ] Contribute to one feature

## üìù First Day Conversation Starters

**Use these if conversation gets awkward:**

1. "What's your coffee order?"
2. "How was your previous job?"
3. "Where are you from?"
4. "What do you like to do outside work?"
5. "What attracted you to us?"
6. "What are you most excited to work on?"
7. "Any questions about the company?"
8. "What's your working style? (morning person? night owl?)"
9. "How do you prefer feedback?"
10. "What's your previous company's best practice we should steal?"

## üéÅ Onboarding Buddy Role

**As an onboarding buddy, you are responsible for:**

- [ ] Daily check-in (How was your day?)
- [ ] Help with technical setup
- [ ] Answer questions (no question too small)
- [ ] Make them feel welcomed
- [ ] Invite them to lunch/dinner
- [ ] Introduce them to team
- [ ] Show them where everything is
- [ ] Check in after week 1 (going well?)

**Onboarding buddy is NOT:**
- Manager (that's manager's job)
- Judge (you're supportive)
- Solo support (manager is also here)

## üìã Manager Checklist

**As manager, you're responsible for:**

### Before Day 1
- [ ] Equipment ready
- [ ] Email/access setup
- [ ] Welcome call scheduled
- [ ] Onboarding buddy assigned
- [ ] Role-specific docs prepared

### Day 1-5
- [ ] Daily check-in (how's it going?)
- [ ] Blockers removed (help them unblock)
- [ ] Culture modeling (show how we work)
- [ ] Make them feel welcomed
- [ ] Role clarity (here's what success looks like)

### Week 2-4
- [ ] Weekly 1-on-1 (15-30 min)
- [ ] Meaningful task assignment
- [ ] Code/work review (if applicable)
- [ ] Feedback (what's going well? what to improve?)
- [ ] Progress check (on track?)

### End of Month
- [ ] Full onboarding review
- [ ] Feedback conversation
- [ ] Career expectations discussion
- [ ] Celebrate! You've onboarded someone!
